NAME

atplan - finger for the ATProto ecosystem

SYNOPSIS

finger <handle>

DESCRIPTION

atplan revives the Unix .plan file tradition for the modern federated web.

In the early days of networked computing, the finger(1) protocol let you query what any user on a connected system was up to. Users maintained a .plan file — freeform text describing their current projects, status, and thoughts. It was proto-blogging before blogging existed.

atplan brings that tradition to ATProto. Publish your plan as a user-owned record on your own PDS. Finger anyone on the network by handle. Your data. Your server. Your plan.

GETTING STARTED

Publish your own .plan from the command line:


$ git clone https://github.com/mikro6/atplan.git

$ cd atplan && python3 -m venv venv && source venv/bin/activate

$ pip install -r requirements.txt

$ cp .env.example .env  # edit with your PDS host

$ python3 plan_write.py


Full documentation at github.com/mikro6/atplan

HISTORY

The finger(1) protocol was defined in RFC 742 (1977) and revised in RFC 1288 (1991). The .plan file tradition flourished on university VAX and Unix systems throughout the late 1980s and 1990s. Notable practitioners included John Carmack, whose .plan files served as a real-time developer journal during the Doom and Quake era.

atplan was built in 2026 as a love letter to that tradition.
